Overview

Workflow Orchestration Engine

Rust runtime with Go control plane workflow engine. Define workflows in YAML using FFL (Forged Flow Language) custom syntax. Replaces Temporal, Apache Airflow, AWS Step Functions, and n8n with a unified, high-performance orchestration layer.

Features

Workflow Capabilities

YAML Workflows

Define workflows in YAML using FFL syntax. Version-controlled, declarative orchestration with built-in validation and IDE support.

Parallel Execution

DAG-based dependency resolution enables automatic parallel execution of independent steps. Maximize throughput with intelligent scheduling.

WASM Plugins

Extend the runtime with WebAssembly plugins. Write custom actions in any language that compiles to WASM for sandboxed execution.

Retry & Error Handling

Configurable retry policies with exponential backoff. Circuit breaker patterns and custom error handlers for workflow resilience.

Cron Scheduling

Built-in cron scheduler for periodic workflows. Time-based triggers with timezone support and missed-run handling.

Workflow Versioning

Semantic versioning for workflows. Run multiple versions simultaneously with automatic migration and rollback capabilities.

Configuration

Workflow Example

# workflow: order-fulfillment
name: order-fulfillment
trigger:
  event: orders.created
  source: forged-stream
steps:
  - name: validate
    action: http.post
    url: ${GATE_URL}/api/v1/orders/validate
  - name: reserve-inventory
    action: http.post
    url: ${GATE_URL}/api/v1/inventory/reserve
    depends_on: [validate]
  - name: notify
    action: stream.publish
    topic: notifications.order
    depends_on: [reserve-inventory]
