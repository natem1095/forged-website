Overview

What is Forged Stream?

Forged Stream is a Rust-based event streaming broker that implements the Kafka wire protocol. Your existing Kafka clients connect unchanged — no code modifications, no new SDKs, no migration complexity. It delivers 500K+ operations per second with persistent storage, consumer groups, and exactly-once delivery semantics.

Replaces: Apache Kafka, AWS Kinesis, RabbitMQ, Confluent Platform

Features

Built for Event-Driven Architecture

Kafka Wire Protocol

Full protocol compatibility with Apache Kafka. Existing clients connect on port 9092 without modification. Drop-in replacement for Kafka brokers.

Persistent Storage

Durable message storage with configurable retention policies. Replay events from any point in time. Zero data loss with fsync guarantees.

Consumer Groups

Balanced partition assignment with automatic rebalancing. Multiple consumers read from the same topic with coordinated offset management.

Topic Management

Create, configure, and manage topics with partition control. Dynamic topic creation and automatic partition rebalancing across brokers.

Schema Registry

Built-in schema validation and evolution for Avro, Protobuf, and JSON. Ensure message compatibility across producer and consumer versions.

Exactly-Once Delivery

Idempotent producers and transactional messaging support. Atomic writes across multiple partitions with rollback capabilities.

Compatibility

Your existing Kafka clients connect unchanged

Forged Stream implements the complete Kafka wire protocol. Point your existing producers and consumers to port 9092 — no code changes required.

# Kafka Producer Config (unchanged)
bootstrap.servers=localhost:9092
key.serializer=org.apache.kafka.common.serialization.StringSerializer
value.serializer=org.apache.kafka.common.serialization.JsonSerializer
acks=all
retries=3

# Works with Forged Stream immediately
producer.send(new ProducerRecord<>("orders.created", order));

Configuration

Simple YAML Configuration

# stream.yaml
broker:
  port: 9092
  storage: /data/stream
  retention: 7d
topics:
  - name: orders.created
    partitions: 12
    replication: 3
  - name: inventory.updated
    partitions: 6
