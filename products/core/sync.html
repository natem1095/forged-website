Overview

Real-Time Data Synchronization

Forged Sync is a high-performance change data capture (CDC) and data synchronization engine that monitors databases, captures changes in real-time, and streams them to downstream systems. It replaces tools like Debezium, AWS DMS, Fivetran, and Informatica with a unified, high-performance solution written in Rust.

With support for 50+ source and sink connectors, Sync captures database changes at the transaction log level and publishes them to Stream Broker for consumption by analytics systems, search indexes, caches, and other downstream applications. Schema transformation, field masking, and checkpoint recovery ensure data quality and reliability.

Capabilities

Core Features

Change Data Capture

Monitor database transaction logs (WAL, binlog, redo) to capture inserts, updates, and deletes in real-time with sub-second latency.

Connector Framework

50+ source connectors (PostgreSQL, MySQL, Oracle, SQL Server, MongoDB) and sinks (Stream, Kafka, S3, Elasticsearch, Snowflake).

Schema Transformation

Rename fields, mask sensitive data, filter columns, and apply custom transformations on-the-fly without impacting source systems.

Checkpoint Recovery

Persistent offset tracking with exactly-once delivery guarantees. Automatically resume from last checkpoint after failures.

Parallel Processing

Partition large tables across multiple workers for high-throughput initial snapshots and ongoing replication.

Dead Letter Queue

Automatically route failed records to a DLQ for manual inspection and replay without blocking the sync pipeline.

Integration

Works With Your Stack

Forged Sync integrates seamlessly with other Forged products to create a complete data platform:

  • Stream — Publishes CDC events to the Stream Broker for real-time consumption by analytics and microservices
  • Store — Captures changes from Store (PostgreSQL) and replicates to downstream systems
  • Lake — Feeds real-time data to Lake for analytics and reporting
  • Flow — Triggers workflows based on database changes (e.g., send email when order status changes)
  • Observe — Monitors sync health, lag, throughput, and error rates
Configuration

Simple YAML Configuration

Define sources, sinks, and transformations in a single configuration file:

# sync.yaml
source:
  type: postgresql
  connection: postgres://localhost:5433/erp
  tables: [orders, inventory, customers]
sink:
  type: forged-stream
  broker: localhost:9092
  topic_prefix: cdc.
transform:
  - table: orders
    mask: [customer_ssn]
    rename: {order_id: id}
