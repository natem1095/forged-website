Overview

Forged Cache is a high-performance distributed cache built in Rust with full RESP3 wire protocol compatibility. It supports every Redis data structure and command, making it a true drop-in replacement for Redis, Memcached, AWS ElastiCache, and Azure Cache for Redis.

The key advantage: your existing Redis clients work unchanged. Whether you're using ioredis, redis-py, Jedis, or any other Redis-compatible client, simply point it to port 6379 and it connects without modification. No code changes, no migration effort, just instant compatibility.

Drop-in Compatibility

Your existing Redis client code works without changes. Connect to port 6379 and start caching.

Node.js (ioredis)

// Your existing code works unchanged
const redis = require('ioredis');
const client = new redis(6379, 'localhost');
await client.set('session:abc', JSON.stringify(user));
const session = await client.get('session:abc');

Python (redis-py)

# Standard Redis client, no changes needed
import redis
r = redis.Redis(host='localhost', port=6379)
r.set('user:1000', json.dumps(user_data))
r.expire('user:1000', 3600)  # TTL support

Java (Jedis)

// Works with any Redis Java client
Jedis jedis = new Jedis("localhost", 6379);
jedis.hset("product:500", "name", "Widget");
jedis.hset("product:500", "price", "29.99");
Map<String, String> product = jedis.hgetAll("product:500");

Features

All Data Structures

Complete support for Strings, Lists, Sets, Sorted Sets, Hashes, and Streams. Every Redis data structure available with full command compatibility.

TTL & Expiry

Per-key time-to-live with both lazy and active expiration mechanisms. Set expiration on any key with automatic cleanup.

Eviction Policies

Multiple eviction strategies including LRU (Least Recently Used), LFU (Least Frequently Used), random, and volatile-ttl for memory management.

Persistence

Optional RDB snapshots for point-in-time backups and AOF (Append-Only File) logging for durability. Configure based on your needs.

Pub/Sub

Full publish/subscribe messaging support. Broadcast messages to multiple subscribers with pattern-based subscriptions.

Cluster Mode

Horizontal scaling with consistent hashing. Distribute data across multiple nodes for increased capacity and throughput.

Integration

Forged Cache integrates seamlessly across the platform:

  • Gate: Response caching for API endpoints with automatic cache invalidation
  • Identity: Session storage for user authentication tokens and permissions
  • ERP/CRM/HCM: Application-level caching for frequently accessed business data
  • Observe: Cache metrics and performance monitoring with real-time visibility

Configuration

Simple YAML configuration with sensible defaults:

# cache.yaml
server:
  port: 6379
  max_memory: 4gb
  eviction_policy: allkeys-lru

persistence:
  rdb:
    enabled: true
    interval: 300s
  aof:
    enabled: false

cluster:
  enabled: false
  nodes:
    - host: cache-01.local
      port: 6379
    - host: cache-02.local
      port: 6379
