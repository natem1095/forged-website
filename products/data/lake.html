Overview

Forged Lake is a columnar analytics engine built with a Rust server and Go control plane. It implements a lakehouse architecture with Bronze (raw data), Silver (cleaned and joined data), and Gold (aggregated KPIs) zones. Designed specifically for manufacturing analytics, Forged Lake provides fast SQL queries over Apache Parquet files with predicate pushdown and column projection.

Forged Lake replaces cloud data warehouses like Snowflake, Databricks, AWS Athena, and BigQuery with an on-premise solution that keeps your manufacturing data under your control while delivering comparable query performance through columnar storage and intelligent query optimization.

Data Architecture

The medallion architecture organizes data into three progressively refined zones:

Bronze Zone

Raw data ingestion layer. Stores unprocessed data from ERP systems, IoT sensors, and external sources in its original format. Serves as the immutable source of truth with minimal transformation. Data retained for 90 days by default.

Silver Zone

Cleaned and joined data layer. Contains validated, deduplicated, and enriched datasets. Implements business logic, data quality rules, and cross-system joins. Powers operational analytics and feeds the Gold zone. Data retained for 365 days.

Gold Zone

Aggregated KPI layer. Pre-computed metrics optimized for dashboard queries. Includes manufacturing-specific KPIs like OEE, scrap rates, on-time delivery, and production throughput. Unlimited retention for historical trend analysis.

Features

Columnar Storage

Apache Parquet format with Snappy compression. Column-oriented layout enables efficient analytics queries that only read required columns. Compression ratios of 10:1 or better on typical manufacturing data. Automatic statistics collection for query optimization.

SQL Analytics

Full SQL query engine with predicate pushdown, column projection, and partition pruning. Push filters and aggregations down to the storage layer to minimize data transfer. Supports JOINs, window functions, and complex analytical queries.

Data Zones

Bronze/Silver/Gold medallion architecture separates raw ingestion from analytics. Progressive refinement through the zones ensures data quality while maintaining lineage. Independent scaling and retention policies per zone.

KPI Tables

Pre-computed Gold tables for common manufacturing KPIs. Sub-second query response times for dashboards. Automated refresh pipelines keep metrics current. Custom KPI definitions through SQL views.

Ingestion Pipeline

Batch and streaming data loading from Forged Sync and Forged Stream. Automatic partition management and compaction. Schema validation and type coercion. Idempotent writes prevent duplicate data.

Schema Evolution

Backward-compatible schema changes without rewriting data. Add columns, modify types, and restructure tables without downtime. Automatic migration of new columns to existing Parquet files on read.

Platform Integration

Forged Lake integrates seamlessly with the Forged platform:

Forged Sync (data ingestion) Forged Stream (streaming ingestion) Forged Store (OLTP source) Forged Observe (pipeline metrics) Forged ERP (manufacturing data) Forged Console (dashboard queries)

Configuration

Configure Forged Lake through a simple YAML file:

# lake.yaml
server:
  port: 8600
storage:
  base_path: /data/lake
  format: parquet
  compression: snappy
zones:
  bronze:
    retention: 90d
  silver:
    retention: 365d
  gold:
    retention: unlimited
