Features

Core Capabilities

Edge Caching

LRU memory + disk tiered cache for fast content delivery with automatic eviction and warming.

WASM Edge Functions

Deploy custom logic at the edge with WebAssembly for request/response transformation and filtering.

Web Application Firewall

Rule-based request filtering and blocking with IP allowlists, rate limits, and pattern matching.

TLS/ACME

Automatic certificate provisioning and renewal via Let's Encrypt with support for custom CAs.

Rate Limiting

Per-client and per-route rate limiting with configurable windows and burst allowances.

Health Checks

Upstream health monitoring with failover to backup origins and automatic recovery detection.

Configuration

Sample YAML Config

edge:
  port: 8700
  cache:
    memory_size_mb: 512
    disk_path: /var/cache/forged-edge
    disk_size_gb: 50
    ttl_default: 3600
  wasm:
    functions_path: /etc/forged/edge/wasm
    enabled: true
  tls:
    acme:
      enabled: true
      email: admin@example.com
      staging: false
  waf:
    enabled: true
    rules:
      - name: block-bad-bots
        pattern: "(bot|crawler|scraper)"
        action: deny
  rate_limit:
    enabled: true
    requests_per_minute: 120
  origins:
    - url: https://backend.example.com
      health_check:
        enabled: true
        interval: 10s
        path: /health
Integration

Works With

Gate

Sits in front of Gate as edge proxy, caching API responses and applying edge functions before upstream routing.

Observe

Exports edge metrics including cache hit rate, request throughput, and WAF block events to Observe.

Identity

Integrates with Identity for edge authentication, validating JWT tokens at the edge before proxying.

Cache

Can optionally use Cache as shared cache layer for multi-node edge deployments with consistent invalidation.
